# Learning: CAV Child Tasks Use Technical Task

## Context
While creating a tracker under E2E story CAV-132724, Jira MCP metadata showed that the CAV project does not offer an issue type named Sub-task.

## Learning
For CAV child work under a Story, use issue type "Technical task" with the required "parent" field set to the parent Story key. Do not also set Epic Link when creating this child task, because Jira rejects that combination with "You can't add sub-tasks to epics"; the child inherits the epic association from the parent Story. The review transition name in this workflow is "Review", not "In Review".

## Future Use
When the user asks for a Jira subtask under a CAV Story, create a "Technical task" via Jira MCP metadata-first flow, set "parent", omit Epic Link, and transition to "Review" when they ask for in-review status.
