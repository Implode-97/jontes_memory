# Learning: Keep Module READMEs Durable

## Context
During Terraform module work in the data platform modules repo, the user called out that README content about test status and development progress read more like a log than useful module documentation.

## Learning
Module READMEs should describe stable behavior, input contracts, provider expectations, validation commands, and durable prerequisites. Avoid phrasing that records development history, current test status, or one-off implementation progress. When test details are needed, frame them passively as validation prerequisites or environment facts, not as a narrative of what was recently moved, fixed, or verified.

## Future Use
When editing IaC READMEs, remove development-log wording and prefer concise documentation that will still be true after the MR merges. Keep specific environment values only when they are actual test prerequisites.
