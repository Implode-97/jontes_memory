# Learning: MR 32 fails in workspace root-bucket cleanup

## Context
MR !32 pipeline `7961794`, job `36244870`, failed after the branch was cleanly restored to master `7fc05e3`. The same job was retried on 2026-07-13 with the same result.

## Learning
The external-location `enabled -> destroyed` error is an intentional negative E2E assertion; four later lifecycle applies succeed. The actual test failure is deferred cleanup: Terraform cannot delete `tratone2e-870-euc1-dev-dbw-root` because it is non-empty. The MR's janitor helper then cannot empty it because it expects a `root_bucket_name` Terraform output that the generated `dbx_workspace` unit does not expose, so both cache recovery destroy and Terragrunt destroy fail.

## Future Use
Do not diagnose the expected lifecycle-guard error as the root cause. Repair the workspace bucket-name contract or cleanup discovery, verify leaked E2E resources, then rerun the pipeline.
