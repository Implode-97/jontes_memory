# Learning: Cross-repo Terragrunt E2E needs an explicit released module source

## Context
The CAV-134270 catalog E2E locally falls back to an adjacent checkout of `data-platform-terraform-modules`, while the GitLab job checks out only the Terragrunt catalog repository.

## Learning
An unconditional sibling-path fallback works in the shared local development folder but is normally absent in isolated GitLab runners. Before the first module registry tag exists, the Terragrunt MR also cannot use its default `0.1.0` registry source. Unless an external CI variable supplies a valid source, the E2E cannot initialize the new wrapper.

## Future Use
Use an explicit cross-repository release sequence: publish a modules RC, pin that exact RC in the Terragrunt branch, and run the catalog E2E against it. Keep a local sibling override only when the path actually exists. CI should fail early with a clear source-resolution error rather than pass a nonexistent local path into Terragrunt.
