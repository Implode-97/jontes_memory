# Learning: Keep Codex pet work in the user folder

## Context
While upgrading the installed Doge pet to the v2 looking-direction format, the working run was initially created inside an IaC project folder. The user explicitly asked to move it to their user folder instead.

## Learning
Codex pet packages and hatch-pet run artifacts should live under the user-level Codex directories, not inside unrelated project repositories. Doge is installed at `/Users/jnyjc2/.codex/pets/doge` with `spriteVersionNumber: 2`; its retained QA run is `/Users/jnyjc2/.codex/pet-runs/doge-v2-20260713`.

## Future Use
For future pet installs, repairs, or upgrades, default the working run to `~/.codex/pet-runs/<run-name>` and package to `~/.codex/pets/<pet-id>`. Avoid creating pet artifacts under the current code repository unless the user explicitly requests project-local storage.
