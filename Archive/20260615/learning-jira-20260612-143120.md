# Learning: CAV parented follow-up tasks

## Context
While creating a follow-up Jira issue for CAV-131446, the project did not expose an issue type named Sub-task.

## Learning
In project CAV, the parented sub-task-like issue type is Technical task. Its create metadata requires the Parent field, and setting customFields.parent to {"key":"CAV-131446"} successfully created a child issue under the story. Jira assignee lookup for this project worked with the username jnyjc2, while the user's email address was rejected by the Jira create tool.

## Future Use
When the user asks for a CAV sub-task or child follow-up issue, first check create metadata, then prefer Technical task if Parent is required and Sub-task is unavailable. Use the parent key through customFields.parent, and assign to jnyjc2 by username unless a concrete account ID is available.
