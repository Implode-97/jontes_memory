# Learning: Keep lifecycle matrices out of shared catalog E2E

## Context
MR !31 in `data-platform-terragrunt-catalog` failed `platform-e2e-test` after `dbx_external_locations` added an active lifecycle matrix to `external_locations_assert`.

## Learning
The shared platform E2E should remain a full-stack converge/destroy smoke path plus feature output assertions. Repeated lifecycle mutations inside the active E2E can trigger extra full `terragrunt stack run apply` calls, rerun the workspace 300-second propagation hooks, and expose generated-unit Terragrunt output EOF behavior before the intended feature guard reports. This mirrors the existing sandbox catalog pattern, where the lifecycle matrix is kept in a draft file and guard semantics are covered in guard tests.

## Future Use
When adding lifecycle behavior for catalog units, put transition and unsafe-delete coverage in guard/unit Terratest suites first. Do not activate repeated full-stack lifecycle matrices in `platform-e2e-test` unless the shared E2E stability/cost tradeoff is explicitly accepted.
