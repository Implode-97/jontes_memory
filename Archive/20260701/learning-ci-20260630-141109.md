# Learning: External locations E2E module source

## Context
MR !31 in `data-platform-terragrunt-catalog` failed `platform-e2e-test` after the `dbx_external_locations` unit was added. The raw GitLab trace showed Terragrunt trying to download a local sibling path for `databricks-workspace-external-s3-locations-wrapper`.

## Learning
For `dbx_external_locations`, do not let the E2E harness silently default `DBX_EXTERNAL_LOCATIONS_MODULE_SOURCE` to `../data-platform-terraform-modules/...` in CI. The catalog pipeline checks out only the catalog repo, while the unit already has a registry source for `databricks-workspace-external-s3-locations-wrapper/databricks?version=0.1.0` and CI provides `TG_TF_REGISTRY_TOKEN`.

## Future Use
When investigating this E2E lane, treat `stat ... data-platform-terraform-modules/modules/databricks-workspace-external-s3-locations-wrapper: no such file or directory` as source-override wiring failure, not a Databricks or AWS apply failure. Prefer explicit local overrides only when the caller sets `DBX_EXTERNAL_LOCATIONS_MODULE_SOURCE`.
