# Learning: exact dependency package lookup implemented locally

## Context
The `data-platform-terraform-modules` repo had a publish-time bug where `rewrite_publish_module_sources.py` could resolve an unchanged child dependency to a wrapper package version because GitLab package lookup by bare module name was too broad.

## Learning
The local branch now changes `ci/scripts/deploy-changed-modules/rewrite_publish_module_sources.py` so fallback dependency lookup uses the exact Terraform module package identity `<module>/<system>` and ignores returned package rows whose `name` does not match exactly. A regression test was added beside the script to cover the `databricks-account-metastore` versus `databricks-account-metastore-wrapper` collision.

## Future Use
When continuing this work, keep the wrapper README-only edits for `databricks-account-metastore-wrapper` and `aws-databricks-workspace-wrapper`; they intentionally trigger republishing corrected wrapper packages without changing Terraform behavior.
