# Learning: catalog E2E metastore wrapper registry mismatch

## Context
During MR !21 in `data-platform-terragrunt-catalog`, the latest `platform-e2e-test` job `34350802` in pipeline `7656877` failed during `terragrunt stack run apply`.

## Learning
The first real failure was not cleanup fallout. The published package `databricks-account-metastore-wrapper/databricks` version `0.3.0` contains `module "metastore"` with `source = "gitlab.collaborationlayer-traton.com/ats-project/databricks-account-metastore/databricks"` and `version = "0.2.0"`, but GitLab's Terraform module registry only had `databricks-account-metastore/databricks` version `0.1.0`. Terraform therefore failed `init` in `eu-central-1/dbcore/.terragrunt-stack/dbx_metastore` with `Error: Unresolvable module version constraint`.

## Future Use
When this E2E or follow-up pipelines fail around `dbx_metastore`, check the module registry publication state before debugging Databricks auth or cleanup. The likely fix is to publish the child `databricks-account-metastore` version required by the wrapper, or republish/pin the wrapper to a child version that exists.
