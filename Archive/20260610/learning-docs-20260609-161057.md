# Learning: live Terragrunt README must mirror active state and targets

## Context
This came from scoring `data-platform-terragrunt-live/README.md` against the actual root CI, `ci/targets.yml`, generated target-pipeline code, active stack files, and `non-prod/root.hcl`.

## Learning
For live Terragrunt repos, README drift around target inventory, CI stages, generated child-pipeline shape, active stack units, and remote state is high-impact. In this case, the README omitted an active `eu-central-1/dbcore` target, omitted the root `bootstrap` stage, simplified generated downstream pipelines, and said remote state was commented out even though `non-prod/root.hcl` had active S3 remote state with lockfile enabled. The useful parts were the sibling-stack generation note and concrete cross-account role-chain prerequisites.

## Future Use
When reviewing live IaC docs, compare every operator-facing claim to the current CI YAML, target inventory, stack declarations, and root backend config. Score stale state/backend or target claims as rework-level issues even when the prose is otherwise concise.
