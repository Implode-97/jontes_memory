# Learning: live CI targets need canonical account keys

## Context
This came from scoring `data-platform-terragrunt-live/ci/targets.yml` for new-maintainer readability, KISS, YAGNI, maintainability, and safe editing.

## Learning
The file is intentionally small and useful as a CI target inventory, but it becomes hard to maintain when its `target` values do not line up with the live `_targets.hcl` keys selected by sibling `_account.hcl` files. For regional `dbcore` rows, `target: dbcore` is readable for job names but hides the canonical keys `dbcore_euc1` and `dbcore_euw3`, leaving maintainers to infer the mapping through `stack_dir` and account selector files.

## Future Use
When reviewing or updating live target catalogs, prefer an explicit canonical account target key or generation from live stack/account files. Penalize duplicate identity fields and any CI-only shorthand that obscures which `_targets.hcl` entry is safe to edit.
