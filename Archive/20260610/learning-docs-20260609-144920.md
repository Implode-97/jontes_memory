# Learning: Keep IaC Docs Honest About Active Wiring

## Context
While scoring the `dbx_budget_policy` Terragrunt catalog README, reviewers found that the document explained the unit contract well but pointed operators at generated stack paths and rollout/E2E wording that were not active in the current root pipeline or stack declarations.

## Learning
For account-wide or safety-sensitive IaC features, documentation should distinguish active wiring from future rollout. Do not present generated `.terragrunt-stack` paths, E2E coverage, or manual apply flows as usable unless the unit is currently declared in the stack and included in active CI. For Databricks budget policies, describe `workspace_ids = []` as account-wide availability across workspaces, not merely "no binding."

## Future Use
When reviewing or writing Terragrunt unit READMEs, verify stack declarations and root CI includes before accepting validation guidance. Replace MR/future-revision notes with durable current-state wording and call out operator prerequisites, live-test behavior, and blast radius explicitly.
