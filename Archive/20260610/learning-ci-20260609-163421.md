# Learning: target-aware live pipeline selector simplicity

## Context
This came from multi-agent scoring `data-platform-terragrunt-live/ci/scripts/generate_target_pipeline.py` for readability, KISS, YAGNI, maintainability, and live CI target-selection safety.

## Learning
Target-aware live pipelines are valuable, but their selector logic should be boring and explicit. Avoid mixing glob engines with different slash semantics, broad shared patterns that can silently select every target, bespoke YAML parsing in multiple scripts, and text-parsed HCL dependency discovery that ignores the declared dependency path. Workspace or target dependency fan-out should come from one explicit source of truth, such as the target catalog, or from the actual configured dependency path.

## Future Use
When reviewing or implementing live Terragrunt CI fan-out, penalize hidden selector conventions more heavily than ordinary script verbosity. Prefer one tested path-matching model, a shared or structured target parser, explicit dependency declarations, fail-closed no-match behavior, and current GitLab child-pipeline status mirroring (`strategy: mirror`) where the instance supports it.
