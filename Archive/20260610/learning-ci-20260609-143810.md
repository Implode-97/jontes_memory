# Learning: Job files are not active CI unless included

## Context
This came from scoring `data-platform-terragrunt-catalog/tests/terratest/guards/dbx_workspace_assignments/dbx_workspace_assignments_guard_test.go` and checking the related `ci/test/workspace-assignments-guard-test.yml` file.

## Learning
When reviewing catalog CI coverage, the existence of a job YAML file is not proof that the job runs. Check root `.gitlab-ci.yml` includes before treating a package, guard, or E2E lane as active. In this case, `workspace-assignments-guard-test.yml` defines a dedicated job, but the root pipeline currently includes only `guard-tests.yml` and `platform-e2e-test.yml`, not the workspace-assignment guard job. That means the coverage should be scored as useful but not fully active root-pipeline coverage.

## Future Use
For CI/doc/test scoring, compare claims against both job definitions and the root include graph. Penalize docs or tests that imply active coverage when the job is only present as an unincluded file, and either recommend wiring the include or clearly marking the coverage as currently standalone/unwired.
