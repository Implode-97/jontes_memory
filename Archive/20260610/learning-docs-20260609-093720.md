# Learning: Separate Active E2E Proof From Deferred Coverage

## Context
While scoring `data-platform-terragrunt-catalog/docs/TESTING.md`, reviewers found that the documented `plan == 0` and sandbox mutation lifecycle proof were described like active E2E guarantees, while the corresponding Go code currently keeps that lifecycle matrix commented out.

## Learning
Testing documentation should not claim an active correctness proof unless the CI path actually runs it. Deferred, commented, or future lifecycle tests can still be valuable context, but they must be labeled as deferred coverage rather than presented as the current safety gate.

## Future Use
When reviewing IaC testing docs, compare every stated guard, E2E stage, and proof step against the active CI includes and uncommented test code. Penalize docs that blur active coverage with planned or dormant checks, especially around destructive cleanup and convergence guarantees.
