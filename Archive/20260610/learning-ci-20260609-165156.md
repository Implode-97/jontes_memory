# Learning: live CI target YAML shape

## Context
This came from reviewing `data-platform-terragrunt-live/ci/targets.yml` for looks, readability, KISS, YAGNI, maintainability, and safety-aware simplification.

## Learning
The file is intentionally small and active-scope-only, but its current shape repeats path-derived facts (`environment`, `region`, `aws_region`, and `stack_dir`) and uses duplicate `target: dbcore` values across regions while live `_targets.hcl` uses unique keys (`dbcore_euc1`, `dbcore_euw3`). `display_name`, `kind`, and explicit `stack_dir` are justified because they keep generated CI labels and job variables readable and safer. The weakest maintainability point is identity drift between CI target names, path identity, and Terragrunt account selectors.

## Future Use
For future reviews of this target catalog, accept the small flat list as a reasonable starting point, but prefer one obvious identity source. Penalize duplicated region/AWS-region fields unless a real distinction exists, and recommend unique target IDs or explicit dependency fields before adding broader schema features.
