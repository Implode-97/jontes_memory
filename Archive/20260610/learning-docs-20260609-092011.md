# Learning: Contributor Commands Must Match CI Or Be Labeled

## Context
This came from scoring the catalog repo's `docs/CONTRIBUTING.md` against the actual GitLab CI guard-test lane and adjacent CI/testing documentation.

## Learning
Contributor docs should avoid copying CI commands unless they intentionally match the active CI lane. If a local command is broader or stricter than CI, the doc should say that explicitly. In this repo, `go test ./...` under guard tests is not equivalent to the active CI command while `dbx_workspace_assignments` is excluded from the shared guard lane. Required CI variable lists should also distinguish variables defaulted by root CI from variables users must provide.

## Future Use
When reviewing IaC contributor docs, compare local command examples and variable lists with the real CI YAML before accepting them. Prefer a compact workflow guide with links to `TESTING.md`, `CI_CD.md`, `OIDC_AUTH.md`, and `RELEASES.md` over duplicated commands that can drift.
