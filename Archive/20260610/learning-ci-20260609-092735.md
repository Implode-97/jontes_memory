# Learning: Auth Runbooks Must Separate Required, Defaulted, And Verified

## Context
This came from scoring the catalog repo's `docs/OIDC_AUTH.md` against the actual platform E2E GitLab job and current GitLab, AWS STS, and Databricks workload identity documentation.

## Learning
OIDC/auth runbooks can justifiably be more detailed than overview docs, but they must distinguish variables supplied by users, variables defaulted by CI, tokens minted by `id_tokens`, and values actually preflighted or logged by the job. Do not imply verification exists just because a variable should be present. For Databricks workload identity, treat `DATABRICKS_CLIENT_ID` as both the workload-identity client ID and any wrapper/deployer ID when the platform reuses it intentionally.

## Future Use
When reviewing CI auth docs, compare required-variable lists and verification steps to the real job script. Flag duplicated local-debug snippets unless one document clearly owns them, and include duration/expiration/credential-source failure modes for STS-based temporary credentials.
