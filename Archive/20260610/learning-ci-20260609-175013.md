# Learning: live CI target catalog identity

## Context
This came from multi-agent scoring `data-platform-terragrunt-live/ci/targets.yml` for readability, KISS, YAGNI, live target-selection safety, and maintainability.

## Learning
A flat active target catalog is a good live CI shape when it keeps stack directories explicit and operator labels visible. The recurring risk is target identity drift: using repeated short names such as `dbcore` across regions while live `_targets.hcl` uses `dbcore_euc1` and `dbcore_euw3` makes `stack_dir` the hidden identity bridge and can break selector or dependency logic that keys by target name. Duplicating `region` as `aws_region` also adds drift unless those fields can intentionally differ.

## Future Use
When reviewing target-aware CI catalogs, keep explicit `stack_dir` and useful display labels. Prefer canonical target/account keys or a clearly documented composite key, derive duplicate region facts when possible, and put dependency fan-out in the catalog if CI selectors otherwise need to scrape HCL or infer relationships.
