# Learning: Workspace assignment README shape

## Context
This came from scoring `data-platform-terragrunt-catalog/units/dbx_workspace_assignments/README.md` for looks, readability, KISS/YAGNI, and maintainability under a strict documentation-review lens.

## Learning
For workspace-assignment unit READMEs, preserve lifecycle safety guidance such as the state table, destroy-pending staging, safe row-removal rule, and guard intent. Penalize docs that expand into long downstream mock-output blocks or local credential/run-command transcripts, especially when those commands are not active root CI coverage or are mislabeled as apply/destroy smoke while they only run local guard/generated-unit tests.

## Future Use
When reviewing similar Terragrunt catalog docs, recommend moving verbose command setup to test/runbook docs, reducing mocks to the minimal consumer contract, and keeping CI coverage wording explicit about active root includes versus standalone job files.
