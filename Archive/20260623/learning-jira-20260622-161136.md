# Learning: CAV-134396 Scope Priorities

## Context
While refining the `sandbox_catalog/epic` planning pack for Jira epic `CAV-134396`, the user clarified which stories are core ingestion-team onboarding versus stretch or conditional platform work.

## Learning
For CAV-134396, direct Woodpeckers/ingestion-team onboarding is the core PI priority. SQL warehouse and instance profile stories should be pure IaC work in both Terraform modules and Terragrunt catalog repos, following compute policy/serverless budget policy patterns and reusing `_org/teams.yml`, `workspace_teams.yml`, and `_region.hcl`. Tagging is core producer-side IaC work per feature; a Databricks cost dashboard is stretch unless PO requires it or it saves major future effort. Product service-principal execution should be production-workspace-only by default. TOOCAN should remain an open enablement/compliance question around CI/CD, CDK-first tooling, and unknown required integrations.

## Future Use
When continuing this epic, rank stories by whether they directly enable ingestion-team onboarding. Treat dashboards, broad prod product-SP refactors, and TOOCAN tooling integrations as conditional unless PO makes them acceptance-critical.
