# Learning: CAV-134396 Story 03 Split

## Context
While refining the `sandbox_catalog/epic` planning pack for CAV-134396, the user identified that Story 03 was too broad and should split workspace assignment and serverless budget policy into separate placeholders.

## Learning
For CAV-134396, keep Story 03 focused on data product service-principal release status, output contract, and workspace-class rules. Add Story 03b for workspace assignment IaC and product execution isolation; this is core because Woodpeckers cannot be onboarded without reliable workspace placement. Add Story 03c for serverless budget policy onboarding alignment; this is conditional core only if serverless usage is part of Woodpeckers or PO acceptance, otherwise stretch. Serverless budget policy should remain separate from Story 08, which owns tagging contract and optional dashboard scope.

## Future Use
When continuing the epic, route workspace placement work to 03b and serverless budget-policy work to 03c. Do not expand Story 03 back into a mixed implementation bucket.
