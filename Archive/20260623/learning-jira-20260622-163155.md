# Learning: CAV-134396 Data Product Branch Is Input, Not Capability

## Context
While correcting the CAV-134396 epic planning pack, the user clarified that the current data product feature branch is not approved or implemented, even though future implementation decisions should use the branch changes as the basis.

## Learning
For CAV-134396 planning, treat the CAV-134270 data product branch as PoC/start work and implementation input only. It must not be assumed merged, released, approved by the Product Owner, or available for downstream product service-principal consumers. Story 02a should gate whether the data product feature proceeds, defers, or becomes stretch; Story 03 should only define downstream product-SP contracts after that approval.

## Future Use
When planning or implementing CAV-134396 stories, keep team onboarding valid without product service principals. Product-SP dependencies in workspace assignment, compute, SQL, cost tagging, external locations, budget policy, and TOOCAN validation should stay conditional until PO approval and a release contract exist.
