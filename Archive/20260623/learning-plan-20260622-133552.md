# Learning: external location plan passed scoring threshold

## Context
The user asked Codex to iterate on the `sandbox_catalog` Databricks Unity Catalog external-location implementation plan using independent scoring agents until the average score reached 90+.

## Learning
The accepted V1 plan keeps `dbx_external_locations` standalone, non-validating, read-only, all-workspaces, one storage credential per external location, and source-AWS-owned. The final plan fixed key review issues: creator grants reference the created external-location module output to preserve Terraform graph ordering, `source_handoff` is only for non-destroyed rows, file events are disabled by default with provided-SQS opt-in, and lifecycle tombstones/guards protect destructive retirement.

## Future Use
When implementing this feature, treat remaining reviewer comments as implementation discipline: keep validation pragmatic, avoid broad outputs, keep README handoff compact, do not add AWS/KMS/SQS management, and keep E2E as a contract smoke rather than S3/IAM readability proof.
