# Learning: Prefer real GitLab job traces over pasted CI excerpts

## Context
This came from improving a Terratest and Terragrunt CI log triage workflow after a root-cause investigation used a user-pasted log excerpt instead of first downloading the actual GitLab job trace. The excerpt was useful, but it was not the authoritative source and that shortcut weakened the investigation.

## Learning
For CI failure investigation in this workspace, the primary source should be the real GitLab job trace fetched directly from the failed job. Pasted excerpts are only fallback evidence when the trace cannot be retrieved. The standard flow should be: identify the exact failed job, download its raw trace, filter the trace to reduce noise, then diagnose from the first retained failure block.

## Future Use
When a user provides CI log text, do not assume that is enough if the underlying GitLab job is accessible. Fetch the actual job trace first, save both raw and filtered logs, and only use pasted excerpts to confirm or supplement the analysis. This should be reflected in CI triage skills and investigation habits.
