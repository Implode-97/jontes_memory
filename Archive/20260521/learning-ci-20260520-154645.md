# Learning: Filter raw CI logs before diagnosing Terratest and Terragrunt failures

## Context
This came from debugging a GitLab E2E pipeline where the important failure lines were buried under normal Terratest, Terragrunt, and Terraform chatter. The user manually filtered the raw log and surfaced the real metastore permission error faster than the earlier investigation.

## Learning
For Terratest and Terragrunt CI failures in this workspace, full raw logs should be downloaded first and then filtered to remove expected noise such as retry chatter, normal `msg=` output, command-start spam, blank `command.go:301` lines, and everything before the first `=== RUN`. The earliest retained failure block usually contains the real root cause, while later cleanup errors are often only fallout.

## Future Use
Use the repo-local `skills/ci-log-triage/scripts/filter_ci_log.py` tool before diagnosing failed jobs. Read the filtered output first, identify the earliest true failure, then go back to the raw log only around that point if additional context is needed.
