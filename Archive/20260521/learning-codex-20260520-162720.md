# Learning: Skills must be explicit and cold-start usable

## Context
This came from refining a CI log triage skill after it relied too much on implied context from the surrounding conversation. The user pointed out that a skill may be started by an agent that does not share the current session context or prior reasoning.

## Learning
For this user, skills should be written as standalone operating instructions for an agent with zero prior context. Implied meaning that might be acceptable in ordinary chat is not sufficient inside a skill. Skills should state their purpose, required inputs, source of truth, workflow, and expected output explicitly.

## Future Use
When creating or updating skills in this workspace, bias toward explicit cold-start instructions. Include concrete retrieval steps, decision rules, and output expectations so a fresh agent can execute the skill correctly without depending on remembered context from the conversation that created it.
