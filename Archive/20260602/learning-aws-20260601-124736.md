# Learning: Prefer Dev AWS Profile

## Context
During Databricks/Terragrunt E2E debugging, AWS SSO was initially authenticated through a broad Administrator SSO profile. The user asked to avoid that pattern and prefer a narrower dev-scoped profile for AWS investigations.

## Learning
Use a dev-scoped AWS profile by default for AWS checks in this project, especially for dmp-dev/IAM/CI debugging. Do not use broad Administrator SSO unless the user explicitly asks for it or the narrower profile cannot answer a required read-only question and the user approves escalation.

## Future Use
Before AWS IAM, STS, CloudTrail, or S3 diagnostics, verify the selected AWS profile and account. Prefer `--profile dmp-dev` or another clearly dev-scoped profile, and state when that profile is not authenticated or lacks access instead of silently falling back to Administrator.
