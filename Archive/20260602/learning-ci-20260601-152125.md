# Learning: Terraform module CI runs all tests files

## Context
During investigation of `data-platform-terraform-modules` MR !34, the dynamic module test runner selected the sandbox wrapper and external-location modules and ran plain `terraform test` for each selected module.

## Learning
In this repo, normal module MR CI includes every `tests/*.tftest.hcl` file in a selected module. A live apply test named `tests/valid_apply_real_provider.tftest.hcl` is therefore part of ordinary MR CI unless CI adds a filter or the file is moved out of `tests/`. The file-events change also exposed that setting `enable_file_events = true` on `databricks_external_location` through Terraform requires a `file_event_queue` block; IAM permissions alone are not enough.

## Future Use
When adding real provider tests to modules, treat placement under `tests/` as CI-enabling. For Databricks file events, either model the queue explicitly, keep file events disabled in Terraform until queue support is implemented, or separate UI/default behavior from Terraform provider requirements.
