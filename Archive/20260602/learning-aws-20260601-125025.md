# Learning: AWS Account Scope Matters Most

## Context
After clarifying AWS profile usage for dmp-dev investigations, the user explained that the key safety boundary is not the exact local profile name but whether the authenticated identity is limited to the dmp-dev AWS account.

## Learning
For this project, prefer a narrower dev-scoped AWS profile when available. If local profiles such as `catalog-deploy` or `scania-bedrock` resolve to Administrator but only inside dmp-dev account `498873881431`, that is acceptable for read-only investigation. What must be avoided is silently using an admin identity that can operate across all AWS accounts.

## Future Use
Before AWS checks, run `aws sts get-caller-identity` with the selected profile and verify the account is `498873881431`. State the account and role in the working notes. Continue to keep investigations read-only unless the user explicitly approves changes.
