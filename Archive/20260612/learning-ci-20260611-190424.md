# Learning: Compute policy E2E dependency evaluation failure

## Context
While triaging the latest platform E2E GitLab jobs for the `data-platform-terragrunt-catalog` compute policy MR, the same first failure appeared in jobs `35161731`, `35161640`, and older context job `35161428`.

## Learning
The failure was not caused by missing `dbx_workspace_assignments` output or a Databricks API problem. The upstream workspace assignment unit applied successfully, then `dbx_compute_policy` failed while evaluating generated stack HCL with `Unknown variable; There is no variable named "dependency"` at direct `dependency.workspace_assignments.outputs.workspace_team_assignments` references and at the provider heredoc using `dependency.workspace.outputs.workspace_url`. Cleanup failures were secondary fallout because destroy re-read the same invalid unit HCL.

## Future Use
For this repo, diagnose platform E2E failures by finding the first apply failure before cleanup. If this signature recurs, focus on Terragrunt dependency evaluation scope in generated stack units and avoid direct dependency interpolation in contexts where stack evaluation does not expose `dependency`; keep live behavior fail-closed.
