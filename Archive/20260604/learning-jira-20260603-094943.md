# Learning: Databricks App Auth Support Story

## Context
During a Databricks App image-loading support investigation, the user asked to create a Jira tracing ticket after recommending explicit forwarded user-token authentication with the Databricks Python SDK.

## Learning
Jira MCP was available after retry and the support Story was created as CAV-133791, summary "Trace Databricks App user-token authentication fix for volume image loading". It was created in project CAV under epic CAV-130891, "Firecrackers Support PI2616", with component "Sys.Firecrackers", active sprint "ATS Pi2616: Sprint 4 w23-24" ID 36973, labels "auth", "databricks-app", and "support-trace", and product owner "spahj5".

## Future Use
For follow-up on this Databricks App authentication issue, reference CAV-133791. The ticket records that bare WorkspaceClient calls in Databricks Apps can pick up app service-principal auth and that user pass-through should construct WorkspaceClient with the forwarded token and auth_type "pat".
