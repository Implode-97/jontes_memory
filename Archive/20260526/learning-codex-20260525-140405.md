# Learning: Track Bug Comments With Owner And Date

## Context
While fixing the sandbox catalog wrapper IAM propagation wait, the user corrected the explanatory Terraform comment style and asked Codex to remember it for future bug or unique-behavior comments.

## Learning
For comments that document bugs, timing-sensitive behavior, provider quirks, or other unusual implementation details in this platform IaC code, use a short tracked format rather than a long explanatory paragraph. The preferred shape is `// jnyjc2 YYYY-MM-DD: concise practical reason.` Example: `// jnyjc2 2026-05-25: time-sensitive dependency on IAM role policy settling.`

## Future Use
When adding comments for non-obvious bug workarounds or unique behavior, keep them concise, include `jnyjc2` and the current date, and state the operational reason. Avoid long comment blocks unless the surrounding code genuinely needs more context.
