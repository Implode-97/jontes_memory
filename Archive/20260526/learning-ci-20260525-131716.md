# Learning: Platform E2E external location IAM propagation

## Context
Catalog MR !21 pipeline `7662567` retry job `34434995` reached the sandbox catalog unit after prior orphaned IAM role and Databricks group blockers were removed.

## Learning
The first apply failure moved to `dbx_sandbox_catalogs`: `databricks_external_location` for `s3://tratondev-dmp-euc1-sandbox-firecrackers` failed with Databricks reporting that the AWS IAM role lacks READ permissions. The raw trace shows the sandbox IAM role policy finished at `11:09:04.971Z` and external location validation started at `11:09:05.011Z`, only about 40 ms later. The plan included `credential_name = "sandbox_tratondev_dmp_euc1_storage"`, so this is not a missing credential input; it is consistent with IAM trust/policy propagation racing Databricks validation.

## Future Use
When this failure recurs, treat cleanup errors as secondary. Fix direction is to add an intentional wait/retry before external location validation or otherwise handle Databricks/AWS IAM propagation, rather than only deleting resources and retrying.
