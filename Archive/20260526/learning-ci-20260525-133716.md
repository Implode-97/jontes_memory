# Learning: Sandbox UC IAM propagation is not covered by module CI

## Context
While investigating catalog MR pipeline job `34434995`, the `dbx_sandbox_catalogs` Terragrunt unit failed creating `databricks_external_location` roughly 40 ms after the sandbox IAM inline policy finished creating.

## Learning
The sandbox wrapper IAM policy shape is correct for Databricks Unity Catalog S3 access: bucket list/location, object read/write/delete, and self-assume permissions are present, and the external location plan has `credential_name` set. The likely root cause is IAM/Databricks propagation, not a missing Terraform dependency or stale module version. The normal modules repo dynamic test runner only executes `tests/*.tftest.hcl`; the sandbox wrapper real apply smoke test lives under `real_tests/`, so standard module CI does not prove the live Databricks external-location validation path.

## Future Use
For this failure class, prefer a module-level readiness wait before external locations, and consider a separate protected/manual/scheduled CI job for sandbox wrapper `real_tests/`.
