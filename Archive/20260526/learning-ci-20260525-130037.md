# Learning: Platform E2E orphaned metastore owner group

## Context
While investigating catalog MR !21 pipeline `7662567`, latest retry job `34434468` failed after the user manually deleted the prior IAM role collision.

## Learning
The next blocker was not AWS IAM. The first retained CI error was `dbx_metastore` failing to create Databricks account group `platform-tratondev-dmp-euc1-metastore-admins` because it already existed. The retry did clean up the newly created team identity groups and the metastore service principal via janitor destroys, but the metastore owner group remained outside Terraform state because its creation failed on the existing-name collision.

## Future Use
For future platform E2E retries in this branch, distinguish the first apply failure from cleanup noise. If this exact group collision appears, clean up or import the orphaned Databricks account group before rerunning. Also remember that cleanup can still report incomplete because workspace and sandbox janitor destroys need unavailable metastore/workspace dependency outputs during destroy.
