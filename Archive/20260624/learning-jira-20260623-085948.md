# Learning: Jira Need Field Formatting Differs From Comments

## Context
While updating CAV-134396 child Stories, the user reported that the custom Jira Need field formatting was wrong after the initial Markdown-based write.

## Learning
For CAV Data Platform Stories, `customfield_11317` ("Need") should be written as Jira wiki-style text such as `h2. Context` headings and `#` ordered-list lines. The usual Jira MCP Markdown shortcut that works for descriptions and comments does not reliably render in this custom field. Conversely, raw `#` list markers passed into Jira comments can be converted incorrectly into `h1.` heading lines by the MCP comment conversion path; comments should still use the local formatting skill's safe Markdown input with `##` headings and `1. ` ordered-list lines.

## Future Use
When writing Story descriptions/comments, use safe Markdown and verify read-back. When writing the Need field directly through `additionalFields.customfield_11317`, use Jira wiki-style field content and verify it no longer contains raw Markdown headings like `##`.
