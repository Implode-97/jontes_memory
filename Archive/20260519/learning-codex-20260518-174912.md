# Learning: Firecrackers epic loop is Jira-driven

## Context
The user created and used a Firecrackers epic loop skill for Data Platform IaC work under epic `CAV-130873`.

## Learning
When the user invokes the Firecrackers epic loop, Codex should inspect the Jira epic, focus on stories whose summaries begin with IaC or are clearly platform IaC work, and choose the next useful story based on backlog, in-progress, or review state. The loop should read Jira descriptions and Need fields as primary context, inspect relevant local repos, implement or review the work, run focused tests, open or update GitLab merge requests, and comment or transition Jira only when the work state actually changed.

## Future Use
Use the repo-local `firecrackers-epic-loop` skill when available. Do not choose work only from local files; Jira status and story content drive prioritization.
