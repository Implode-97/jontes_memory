# Learning: IaC MR reviews combine local skills and story context

## Context
The user created and used an MR review workflow for Data Platform repos, including colleague MRs in modules, catalog units, and live IaC repos.

## Learning
For IaC MR review, use both the repo-local MR review skill and the user's IaC code style. Fetch or inspect the MR branch, read the related Jira story or ADR, compare the implementation against existing unit/module patterns, and prioritize concrete correctness, safety, CI, migration, and test gaps. The user wants findings first, with clear severity and file references, not a broad rewrite of the MR. Scope changes should be checked against Jira and existing platform design before being accepted as intentional.

## Future Use
When asked to review a Data Platform MR, do not stop at diff inspection. Combine GitLab MR data, Jira context, local patterns, and the IaC style skill before reporting actionable findings.
