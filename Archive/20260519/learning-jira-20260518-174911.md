# Learning: Data Platform Jira stories use Firecrackers defaults

## Context
Multiple sessions created or discussed Jira stories for Data Platform and Firecrackers work, especially under epic `CAV-130873`.

## Learning
For CAV Data Platform stories, the user expects standard Firecrackers defaults unless they override them: project `CAV`, issue type `Story`, component `Sys.Firecrackers`, parent epic often `CAV-130873`, and values borrowed from a known-good reference story such as `CAV-131342` when sprint, component, or product owner need to be resolved. Short story descriptions should be readable and concise, while detailed implementation context belongs in the Need field.

## Future Use
When creating Jira stories, fetch create metadata first, reuse the Data Platform story skill if available, resolve a future sprint carefully, and verify the created issue afterward. Do not paste raw markdown plans directly into Jira fields.
