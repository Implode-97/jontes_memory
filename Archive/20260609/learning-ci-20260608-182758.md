# Learning: Prefer `strategy: mirror` for GitLab child-pipeline gates

When reviewing GitLab dynamic child pipelines, treat `trigger: strategy: depend` as readable-but-stale unless the GitLab instance cannot use `mirror`. Current GitLab docs say `strategy: mirror` mirrors downstream pipeline status exactly and is recommended over `depend` for trigger jobs that should gate later stages.

For bridge jobs that only need ordering before child pipeline creation, set `needs.artifacts` according to actual bridge-job consumption. If child jobs fetch parent artifacts with `needs:pipeline:job`, avoid making the parent trigger job look like it consumes those artifacts directly.
