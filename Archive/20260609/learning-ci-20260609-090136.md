# Learning: Give Live E2E Credentials Cleanup Headroom

## Context
While scoring `ci/test/platform-e2e-test.yml`, the job used GitLab OIDC and AWS STS correctly, but requested a 3600-second AWS session while the Go E2E timeout was also 60 minutes.

## Learning
Live deploy CI lanes need credential lifetime beyond the nominal test timeout so deferred cleanup still has valid credentials. STS/OIDC bootstrap should parse structured JSON, validate expiration, and preflight all required cloud/provider tokens before starting the expensive test. Dense credential bootstrap logic is better kept in a small checked-in helper than embedded as long YAML shell.

## Future Use
When reviewing E2E CI jobs, reward serialized resource groups and OIDC over static secrets, but check for cleanup headroom, structured credential parsing, explicit token validation, and whether the job waits on all active prerequisite guard lanes.
