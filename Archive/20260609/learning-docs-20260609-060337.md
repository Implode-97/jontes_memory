# Learning: Keep workspace catalog README concise but explicit

## Context
During a read-only documentation review of `data-platform-terraform-modules/modules/databricks-workspace-catalog/README.md`, the user asked for a Databricks Unity Catalog workspace-binding lens focused on open versus selected workspace access, automatic provider workspace binding, replacement behavior, grants, and caller/provider responsibilities.

## Learning
The README's core contract is strong when it says empty `workspace_ids` means open/all workspaces, non-empty means exactly selected workspaces with read-write bindings, `provider_workspace_id` must be one selected workspace, automatic provider-workspace binding is intentionally unmanaged, and anchor changes replace the catalog to avoid stale automatic access. The main readability cleanup is not more detail, but reducing the dense auto-binding paragraph.

## Future Use
For future reviews of this README or adjacent catalog-binding docs, preserve the contract-focused shape and avoid tutorial creep. Suggest only small readability improvements, such as splitting the dense binding paragraph or tightening environment-placement examples, unless the implementation contract changes.
