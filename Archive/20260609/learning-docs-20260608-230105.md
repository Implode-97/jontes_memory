# Learning: Metastore wrapper README needs split test lanes

## Context
During a read-only scoring review of `data-platform-terraform-modules/modules/databricks-account-metastore-wrapper/README.md`, the user asked for a strict KISS/YAGNI documentation review focused on readability, operational noise, duplicate detail, and clear live-test lanes.

## Learning
The README shape is mostly useful when it keeps the wrapper purpose, derived names, account-scoped provider requirement, WIF environment variables, and shared-resource safety warning concise. The main maintainability and safety risk is documenting plain `terraform test` as the routine command while a real-provider apply test lives under default `tests/`, because local users and MR CI can run live shared-resource tests unintentionally.

## Future Use
For future Databricks wrapper README reviews, preserve concise safety/auth notes but push for explicit separation between mock/local checks and opt-in real-provider smoke tests. Prefer moving live tests to `real_tests/` or documenting an exact protected/manual lane; do not let broad child-output examples or pinned shared-resource values dominate the normal usage path.
