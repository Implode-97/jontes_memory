In the ng-iac Terraform modules repo, package-version computation should not rely on GitLab `package_name` as an exact filter. GitLab documents project package `package_name` as fuzzy and the list endpoint is paginated by default, so version code should page results and then exact-match the expected Terraform module package identity, typically `<module>/<system>`, before using versions.

Legacy Bash CI scripts under `ci/scripts/legacy/` are documented as comparison/reference, but when they reimplement active Python CI logic they become drift surface. Prefer deleting them or turning them into tiny wrappers around the Python source of truth, especially for release/versioning, Git diff detection, package lookup, and artifact generation.
