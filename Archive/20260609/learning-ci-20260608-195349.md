# Learning: live Terraform tests in default test path

## Context
During the multi-agent scoring sweep, the sandbox catalog wrapper's `valid_apply_real_provider.tftest.hcl` was reviewed together with the dynamic module test runner.

## Learning
In `data-platform-terraform-modules`, the CI runner executes plain `terraform test` for selected modules, so any real-provider apply file under `tests/*.tftest.hcl` becomes part of ordinary dynamic module CI. This can be justified for high-value smoke coverage, such as proving Databricks can validate a real storage credential/external location against AWS, but it should be treated as a live shared-environment test with cleanup and auth risk.

## Future Use
Keep real apply tests in `tests/` only when normal MR CI is intentionally expected to create and destroy live resources. Otherwise move them to an opt-in/protected path. Document required AWS/Databricks OIDC/auth variables, shared fixture IDs, stale-resource cleanup or janitor steps, and any test-only lifecycle exception such as starting directly in `destroy_pending`.
