# Learning: Package Exclusions Need Active Replacement Lanes

## Context
While scoring `ci/test/guard-tests.yml`, the shared guard job excluded `dbx_workspace_assignments` with `go list ./... | grep -v`, but the dedicated workspace-assignment guard job file was not included by root `.gitlab-ci.yml`.

## Learning
`go test $(go list ./... | grep -v ...)` is not equivalent to `go test ./...`; it intentionally removes package coverage. That can be acceptable only when a separate active CI lane covers the excluded package and the path triggers/docs match that split. A job file sitting unused in `ci/` does not count as coverage.

## Future Use
When reviewing CI test commands, verify exclusions against the root CI includes and path rules. Penalize hidden shell filters when explicit package lists, named jobs, or documented active split lanes would make coverage easier to reason about.
