# Learning: repo-local skill files score best with lean metadata

## Context
During the multi-agent scoring sweep of `ng-iac` repositories, the first reviewed files were repo-local Codex skills for Terraform module work, including IAM role and secure S3 bucket skills.

## Learning
The repo-local module skills are useful and generally score well when they preserve explicit module contracts, safety invariants, and test expectations. The recurring readability loss comes from overloaded frontmatter descriptions, overview text that repeats metadata, missing concrete examples for validation rules, and unclear boundaries between public module outputs and test-only diagnostic outputs.

## Future Use
When creating or updating repo-local skills, keep frontmatter concise but trigger-rich, avoid repeating it in the body, and use small concrete examples for naming or JSON validation rules. Preserve explicit security baselines and destructive-action guardrails, but label live-provider tests and diagnostic outputs clearly so future reviewers do not mistake test convenience for public API design.
