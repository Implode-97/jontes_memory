# Learning: CI guides must track the actual root pipeline

## Context
During the multi-agent scoring sweep, `data-platform-terraform-modules/ci/README.md` was reviewed against the current `.gitlab-ci.yml`.

## Learning
Comprehensive CI guides are valuable for dynamic child pipelines, artifact handoff, registry publishing, Codex review, and OIDC authentication, but they become risky when they drift from the root pipeline. The reviewed guide omitted the current `static-check` stage/file/job path while documenting the rest of the pipeline in detail. It also repeated `modules/**/*` gating and Codex review behavior across several sections.

## Future Use
When editing CI docs, verify the stage list, included files, jobs, artifacts, and troubleshooting sections against the current `.gitlab-ci.yml` and CI fragments. Prefer one authoritative source-gating section, one concise Codex threat-model note, and links to conventions docs for logging/file-ID rules instead of repeating the same policy in multiple places.
