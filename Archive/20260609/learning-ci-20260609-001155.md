# Learning: Serverless budget policy live test should be opt-in

## Context
During a read-only scoring review of `data-platform-terraform-modules/modules/databricks-account-serverless-budget-policy/tests/valid_apply_real_provider.tftest.hcl`, the user asked for a pragmatic CI/platform maintainer view focused on live apply cost, default `tests/` placement, and minimum assertions.

## Learning
For the serverless budget policy module, a real Databricks account apply can be useful only as an explicit live smoke lane. The valuable signal is narrow: real account provider authentication, actual account-level budget policy creation, empty workspace binding behavior, and ACL rule creation for both no-manager and manager-backed policies. Assertions that preserve broad `output.policies` fields, raw caller echoes, tags, or `policy_names` should be treated as maintenance debt unless those outputs are confirmed consumer contracts.

## Future Use
When reviewing or editing this module's tests, prefer mock tests in default `tests/` and move live applies to `real_tests/` or a protected/manual filtered lane. If the live test remains, keep only narrow ID, binding, and ACL assertions tied to real-provider behavior.
