# Learning: GitLab MR note writers need explicit write credentials

When reviewing CI helpers that create GitLab merge request notes, do not treat `CI_JOB_TOKEN` as a normal write credential unless the GitLab instance explicitly supports it. Current GitLab CI job-token docs list Notes API access for reading MR notes, while creating MR notes uses the Notes API `POST` endpoint and normally needs a token accepted through `PRIVATE-TOKEN`.

Good MR-note helper shape: prefer the configured bot/private token write path, add a network timeout, pre-check GitLab's note body size limit, emit a clear missing-token error, and avoid duplicate note spam when jobs can be retried.
