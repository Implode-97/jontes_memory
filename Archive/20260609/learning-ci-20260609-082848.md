# Learning: Bound GitLab API Helpers

## Context
While scoring `check_auto_review.py`, the script was readable but used the GitLab MR Notes API without pagination or request timeouts.

## Learning
Small CI helper scripts should still treat GitLab REST list endpoints as paginated APIs. `per_page=100` is not a complete scan, and missing pagination can silently break guard semantics on busy merge requests. Network calls should also set explicit timeouts, and CI logs should avoid dumping raw API response bodies unless the content is known to be safe and useful.

## Future Use
When reviewing Python CI helpers around GitLab APIs, check for pagination, timeout, bounded logging, and clear token-selection behavior before scoring them as robust. Fail-open behavior can be acceptable for optional advisory lanes, but the failure mode should be intentional and visible.
