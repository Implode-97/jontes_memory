# Learning: Keep Terraform README test commands mock-safe

## Context
While scoring the `aws-databricks-workspace-wrapper` README, reviewers found that the docs showed a plain `terraform test` command even though the module also stores real plan/apply smoke tests under the default `tests/` directory.

## Learning
When a Terraform module mixes mock tests and real-provider smoke tests in the same test tree, README quick-start commands should default to mock-safe `-filter` usage or clearly label the command as CI/live-resource behavior. A plain `terraform test` instruction can mislead maintainers into running live applies or accepting default CI behavior without noticing shared-resource, credential, and cleanup implications.

## Future Use
When reviewing module docs, cross-check every README test command against the actual `.tftest.hcl` files. If live tests are present under `tests/`, recommend mock-only filters for local quick validation and separate explicit live smoke commands with resource and cleanup caveats.
