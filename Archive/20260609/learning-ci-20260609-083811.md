# Learning: Detect Breaking Footers From Full Commit Text

## Context
While scoring `detect_bump_type.py`, reviewers found that release bump detection could prefer `CI_COMMIT_TITLE` over `CI_COMMIT_MESSAGE`, causing `BREAKING CHANGE` footers in the full commit body to be missed.

## Learning
Release bump helpers should treat breaking-change markers as safety-critical. Conventional commit title parsing is useful, but the full commit message must still be scanned for `BREAKING CHANGE` footers before deciding a minor or patch bump. A compact implementation is not enough if the fallback order can under-bump a release.

## Future Use
When reviewing CI release scripts, check that bump inference is conservative: fail on missing prerequisite artifacts, centralize accepted bump labels, log the observed source of the bump, and always inspect full commit text for breaking-change footers.
