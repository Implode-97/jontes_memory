# Learning: Unsandboxed Codex CI wrappers need exact safety boundaries

When reviewing CI wrappers that call `codex exec --dangerously-bypass-approvals-and-sandbox`, treat a temporary worktree as insufficient isolation by itself. The bypassed Codex process can still use the runner filesystem, network, inherited configuration, and forwarded credentials.

Prefer exact environment allowlists over broad prefixes such as `CI_`, `AWS_`, `GIT_`, `GLAB_`, `NODE_`, or `NPM_`. Add `--ephemeral` and `--ignore-user-config` where the CLI supports them, document the external-sandbox precondition, and ensure temp worktrees are cleaned up even when process launch fails.
