# Learning: root repo READMEs need minimal orientation, not just policy notes

## Context
During the multi-agent scoring sweep of `data-platform-terraform-modules`, the root README was reviewed. It was concise and useful for release-bump and CI-skip behavior, but sparse as an entrypoint.

## Learning
Compact root READMEs are valuable, but they should still answer the first contributor questions: what the repo is for, where primary code lives, where detailed module docs live, and how to start a routine change. Policy notes such as branch prefix release bumps and CI skip behavior are useful, but they do not replace orientation.

## Future Use
When creating or editing root repository docs, add a short purpose sentence, a tiny layout section, and a 3-5 bullet contributor quick start. Delegate deep CI or module details to focused docs, but make the root README navigational enough that a new contributor can find the right next file without tribal knowledge.
