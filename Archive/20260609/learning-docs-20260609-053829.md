# Learning: Keep workspace catalog grant README contract-focused

## Context
During a read-only KISS/YAGNI documentation review of `data-platform-terraform-modules/modules/databricks-workspace-catalog-grant/README.md`, the user asked for scoring against a tiny Terraform module contract.

## Learning
For this leaf module, preserve concise safety distinctions: workspace-level Databricks provider, single-principal `databricks_grant` rather than authoritative `databricks_grants`, strict underscore catalog naming, and caller-owned workspace binding ordering through `depends_on`. A plain `terraform test` README command is acceptable here because the default `tests/` directory currently contains only mock-provider tests, not live provider smoke tests.

For future reviews of this README, do not push it toward a tutorial.
