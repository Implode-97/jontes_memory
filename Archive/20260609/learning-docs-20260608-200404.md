# Learning: workspace wrapper README test guidance needs explicit filters

## Context
During a read-only scoring review of `data-platform-terraform-modules/modules/aws-databricks-workspace-wrapper/README.md`, the README was compared against the wrapper source, tests, and dynamic module CI runner.

## Learning
The README mostly matches the wrapper input, output, and resource surface, but its test section is easy to misread. It shows plain `terraform test` while real provider plan and apply smoke tests live under `tests/`, and the dynamic module CI runner also executes plain `terraform test -no-color`. This means real tests are included unless filtered, so docs should distinguish mock-safe local validation from live shared-environment smoke tests and state that the apply smoke creates a real Databricks workspace and AWS resources.

## Future Use
For future reviews or edits of this wrapper README, prefer explicit mock-test filter commands as the safe default, with real smoke tests documented separately as manual/protected or intentional CI coverage, including credential, pinned fixture, cleanup, and live-resource caveats.
