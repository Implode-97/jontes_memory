# Learning: Team identity real apply tests need opt-in execution

## Context
During a read-only scoring review of `data-platform-terraform-modules/modules/databricks-account-team-identities-wrapper/tests/valid_apply_real_provider.tftest.hcl`, the user asked for a Terraform test/CI lane safety persona focused on default `terraform test` behavior and cleanup risk.

## Learning
The file is short and can prove live Databricks account group creation/nesting, but its placement under default `tests/` makes it part of ordinary module discovery. In this repo, the dynamic child runner checks for `tests/*.tftest.hcl` and runs plain `terraform test -no-color`, so a `command = apply` Databricks account test there is an intentional CI lane decision, not a harmless local smoke file.

## Future Use
For future reviews of this wrapper, score the live test lower unless it is moved to an opt-in `real_tests/` path or normal CI is explicitly filtered/protected. Keep mock apply coverage in default `tests/`, and require live apply tests to document fixture ownership, auth, cleanup/preflight guidance, and live-only assertions.
