# Learning: Validate publish plans before persistent registry writes

When scoring CI scripts that publish immutable or persistent package artifacts, prefer a two-phase shape: first validate every module/package/version and build a publish plan, then perform uploads. Interleaving validation and upload creates partial persistent state when a later item fails.

For GitLab Terraform module publishing, check module/system/version constraints and semver before constructing the `/packages/terraform/modules/.../file` URL. Avoid loose JSON coercion such as `str(value)` for version fields; malformed detect artifacts should fail with clear scripted errors before any upload starts.
