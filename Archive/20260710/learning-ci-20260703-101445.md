# Learning: E2E no-op apply probe marker pattern

## Context
While debugging repeated `terragrunt stack run apply` EOFs in `data-platform-terragrunt-catalog`, the platform E2E was changed so normal output assertions run before the diagnostic no-op root apply.

## Learning
The diagnostic probe copies only the suspect unit templates into the copied E2E example tree under `_e2e_probe_units`, patches the copied `terragrunt.stack.hcl` files to source those probe units, and injects two apply-only Terragrunt `before_hook` markers. `I0720` means the unit reached the hook phase before guard scripts; `I0721` means existing guard hooks completed and Terraform should start next. The final no-op apply sets `E2E_TG_HOOK_PROBE_PHASE=final_noop_apply`, `TG_PARALLELISM=1`, debug logging, no color, and per-unit summaries.

## Future Use
For similar stack EOF investigations, keep probes in the copied E2E tree rather than source units. Read CI logs by checking whether final-noop `I0720`/`I0721` appears for each failed unit before blaming guards, Terraform providers, or Terragrunt pre-command evaluation.
