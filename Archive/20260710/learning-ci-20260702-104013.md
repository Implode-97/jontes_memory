# Learning: second platform E2E root apply fails before external-location mutation

## Context
While triaging GitLab pipeline 7902572 for `data-platform-terragrunt-catalog`, the failed `platform-e2e-test` job was inspected with the `ci-log-triage` flow and raw trace filtering.

## Learning
The pushed diagnostic commit added an unconditional second `terragrunt stack run apply` in `TestPlatformE2E`. The first root apply succeeded. The second root apply failed before any external-location lifecycle YAML mutation ran, with multiple EOF errors and a downstream early exit after `dbx_workspace` reran its `after_hook` propagation wait. This rules out the external-location YAML edit as the immediate trigger for that pipeline failure. The failure also did not show GitLab job concurrency or Terraform state-lock/resource-conflict evidence.

## Future Use
When debugging this E2E area, treat repeated full root stack applies as a separate instability vector. Prefer gating diagnostic no-op applies, using `plan` for convergence checks where possible, and using `TG_PARALLELISM=1` reruns only to isolate Terragrunt run-queue concurrency.
