# Learning: copy linked worktrees carefully

## Context
While creating a short review branch for `data-platform-terragrunt-catalog`, the `data-platform-terragrunt-catalog-hotfix` folder was a linked worktree with a `.git` file pointing at shared worktree metadata under the main repo. A literal `cp -a` copied that pointer instead of creating independent Git metadata.

## Learning
For these catalog hotfix folders, do not assume a copied directory is an independent repo. Check whether `.git` is a file before running branch, reset, or checkout commands in the copy. If `.git` is a linked-worktree pointer, use a standalone clone with `--no-hardlinks` from the local repo and then set `origin` to the GitLab URL.

## Future Use
When asked to copy a hotfix repo into a separate review folder, verify clone isolation first with `ls -ld .git`, `git rev-parse --git-dir`, and `git status` in both folders before mutating branches.
