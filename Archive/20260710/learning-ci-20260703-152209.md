# Learning: targeted generated-unit no-op E2E probe

## Context
During repeated `data-platform-terragrunt-catalog` platform E2E EOF investigation, commit `ce1e2d7` added a diagnostic chain after the normal full apply and output assertions.

## Learning
The E2E now probes repeated apply behavior before the final root `run --all apply`: direct generated-unit no-op apply for `dbx_external_locations`, direct generated-unit no-op apply for the dev `dbx_workspace`, then direct `dbx_external_locations` again. These are Terragrunt single-unit applies run from generated unit directories, not Terraform `-target`. The intended branch signal is whether external locations fails alone, only after workspace reapply, or only under the final root orchestration path.
