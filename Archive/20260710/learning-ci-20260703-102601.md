# Learning: Direct external-location hook probe supersedes copied units

## Context
During `data-platform-terragrunt-catalog` E2E EOF debugging, the earlier copied `_e2e_probe_units` hook injection pattern was rejected because it was too indirect to prove the real feature unit behavior.

## Learning
The active diagnostic design now adds the probe directly to `units/dbx_external_locations/terragrunt.hcl`, with a real script at `units/dbx_external_locations/scripts/e2e-probe-hook.sh`. The direct hooks bracket the external-location guard hooks: `I0720` before the guards and `I0721` after the guards, just before Terraform should start. The script prints only when `E2E_TG_HOOK_PROBE_PHASE` is set, keeping ordinary applies quiet.

## Future Use
Ignore or supersede the previous copied-probe-unit memory for this branch. When triaging the next pipeline, search for direct `dbx_external_locations` markers in initial and final no-op phases before inferring guard, Terraform, or Terragrunt pre-hook failure boundaries.
