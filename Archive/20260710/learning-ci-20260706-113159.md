# Learning: E2E State Bucket Bootstrap Needs s3 TagResource

## Context
While investigating `data-platform-terragrunt-catalog` MR !31 E2E failures, CloudTrail for job `35984713` showed repeated failed `CreateBucket` calls from `traton-dmp-oidc-catalog-deploy-role`.

## Learning
The visible Terragrunt error was `GetObject ... NoSuchBucket`, but CloudTrail showed the real first failure: `CreateBucket` was denied because the role lacked `s3:TagResource` on `arn:aws:s3:::traton-dmp-498873881431-data-platform-tg-catalog-e2e-tfstate`. The role already allowed `s3:CreateBucket`, `s3:PutBucketTagging`, bucket configuration actions, and object access, but Terragrunt v1.1.0 sends S3 bucket tags in the `CreateBucketConfiguration.Tags` request, which requires `s3:TagResource` during creation.

## Future Use
For S3 backend bootstrap failures with creation-time tags, check CloudTrail before relying on Terragrunt's masked error. Add or verify `s3:TagResource` alongside `s3:CreateBucket` and `s3:PutBucketTagging`; `PutBucketTagging` alone is not sufficient for create-with-tags.
