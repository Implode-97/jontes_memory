# Learning: catalog CI pins Terragrunt through release binary

## Context
While investigating repeated platform E2E EOF failures in `data-platform-terragrunt-catalog`, the CI Terragrunt version was bumped from the bundled `alpine/terragrunt:tf1.14.5` binary to Gruntwork Terragrunt `v1.1.0`.

## Learning
The `alpine/terragrunt` Docker tags are primarily Terraform-version-oriented, so changing from `tf1.14.5` to another image tag is not a precise way to test a specific Terragrunt release. The repo now uses a global `TERRAGRUNT_VERSION` variable and a checksum-verified `ci/scripts/install-terragrunt.sh` helper to install the exact Terragrunt release in hclfmt, guard-test, and platform E2E jobs while preserving the existing base image.

## Future Use
For future Terragrunt CI version tests in this repo, update `TERRAGRUNT_VERSION` first and keep the installer helper checksum flow intact. Do not infer the Terragrunt version from the `alpine/terragrunt` Terraform image tag alone.
