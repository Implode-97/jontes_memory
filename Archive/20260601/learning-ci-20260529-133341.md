# Learning: Sandbox wrapper 0.1.0 CI evidence and 30s wait caveat

## Context
During a read-only CI investigation for the sandbox catalog work, the modules repo and Terragrunt catalog pipelines were audited to verify whether the renamed sandbox wrapper real test actually ran.

## Learning
`aws-databricks-workspace-catalog-wrapper-sandbox/aws@0.1.0` was published from modules pipeline `7700847` at commit `269875d3`, after child pipeline `7700883` job `34619943` ran `dist/modules/aws-databricks-workspace-catalog-wrapper-sandbox/tests/valid_apply_real_provider.tftest.hcl` successfully with real AWS and Databricks providers. Terragrunt catalog `dbx_sandbox_catalogs` was pinned to that package version. However, the Terragrunt E2E trace showed `time_sleep.wait_for_sandbox_iam_propagation` did run for 30s before external location validation still failed with S3 READ denied for `tratondev-dmp-euc1-sandbox-firecrackers`.

## Future Use
Do not treat the passing module real test as proof that a 30s IAM wait solves all sandbox E2E cases. It proves the renamed wrapper test ran and package/version matching is not the likely culprit; investigate environment or contract differences next.
