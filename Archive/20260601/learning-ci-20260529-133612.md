# Learning: Module Smoke Tests Do Not Prove Terragrunt E2E Topology

## Context
During sandbox catalog debugging, the modules repo real apply test passed with a 30 second IAM propagation sleep while the Terragrunt catalog E2E still failed on Databricks external location READ validation.

## Learning
The modules wrapper real test and catalog E2E are materially different. The module test uses a pre-existing eu-west-1 workspace/metastore and randomized temporary names, while catalog E2E uses a freshly created eu-central-1 workspace/metastore and deterministic sandbox IAM role, storage credential, and bucket names. A passing module smoke test proves the module can work in that fixture, but does not prove the registry package is reliable for the live Terragrunt topology.

## Future Use
When debugging this area, compare environment topology, name reuse, workspace/metastore freshness, and exact package consumption before concluding that a fixed sleep solved IAM/Databricks readiness. Treat the shared E2E pipeline as the authoritative validation for this contract.
