# Learning: Catalog E2E Reuses Deterministic Sandbox Names

## Context
During the CAV-132831 / data-platform-terragrunt-catalog MR !21 investigation, platform-e2e-test failed creating a Databricks external location even though the wrapper package contained and executed the 30s IAM propagation sleep.

## Learning
The modules repo real apply test is not equivalent to the catalog E2E topology. The module test uses a pre-existing eu-west-1 workspace/metastore and randomized naming prefixes/team keys, so each run gets fresh IAM, bucket, storage credential, and external location names. The catalog E2E uses deterministic live names such as `tratondev-dmp-euc1-sandbox-uc-role`, `sandbox_tratondev_dmp_euc1_storage`, and `tratondev-dmp-euc1-sandbox-firecrackers` while also creating workspace/metastore resources in the same stack.

## Future Use
When diagnosing similar CI failures, verify whether a passing module real test used randomized isolated names before treating its propagation delay as proof for deterministic Terragrunt live units. Prioritize name reuse, just-created workspace/metastore readiness, and provider-side caching over assuming the sleep was absent.
