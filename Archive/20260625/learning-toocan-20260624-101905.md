# Learning: TOOCAN Terraform feasibility is a decision gate

## Context
The user asked whether their Data Platform IaC work can become "TOOCAN compliant" while remaining Terraform + Terragrunt based, because the visible TOOCAN guidance appears CDK-focused.

## Learning
Developer Portal/Capy findings: the indexed TOOCAN docs do not define a single formal "TOOCAN compliant" certification checklist. They describe TOOCAN as a golden path centered on standardized CDK constructs, a CDK CI component, cdk-nag AwsSolutions infrastructure policies, GitLab security policies, CI/CD security checks, and automated dependency updates. The IT Security Assessment Guide says TOOCAN controls are supporting evidence and risk-reduction context; teams still own scope, method, findings, and risk decisions. No explicit Terraform/Terragrunt acceptance path was found.

## Future Use
Treat TOOCAN work in this repo as a product/platform-owner decision gate before implementation. For Terraform/Terragrunt, build a control-equivalence gap map and get TOOCAN/support sign-off before adding platform code.
